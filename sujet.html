Mini serveur web dynamique

Sujet du TP

Le serveur web à réaliser est une extension du serveur MiniWeb. Sa particularité est de permettre à des applications java de s’exécuter sous le contrôle du serveur. Les pages html pourront contenir des référence à des classes java sous forme de liens « clickables ». Lorsque l’utilisateur suit un tel lien, la classe est chargée dans le serveur et une méthode nommée : init() est exécutée. Cette méthode doit retourner une chaîne de caractères (String java) qui s’affichera comme une page html sur le navigateur.

 

Les classes que le serveur peut charger doivent implémenter l’interface webclasses.MiniServlet qui ne contient que la méthode init(). Si les classes sont stockées dans des sous-répertoires, il faut qu’elles fasse partie d’un package java reprenant l’arborescence. Par exemple, la classe DateServeur est dans le sous-répertoire webclasses du répertoire courant (celui d’où est lancé le serveur), elle fait donc partie du package webclasses (source du code webclasses\DateServeur.java). Dans une page html référençant une classe, le lien doit être fait avec le chemin permettant de trouver le fichier et correspondant au nom du package et de la classe. Par exemple le lien classe DateServeur montre que l’on appelle la classe DateServeur dans le sous-répertoire webclasses.
